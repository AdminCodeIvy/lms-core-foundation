Key Files to Understand:
========================

1. Entry Point:
   src/server.ts → Starts the server

2. App Setup:
   src/app.ts → Configures Express

3. Routes:
   src/routes/v1/index.ts → All API routes

4. Example Module (Customer):
   src/services/customerService.ts → Business logic
   src/controllers/customerController.ts → Request handling
   src/routes/v1/customerRoutes.ts → Route definitions

5. Authentication:
   src/services/authService.ts → Auth logic
   src/middleware/auth.ts → JWT verification

6. WebSocket:
   src/websocket/socketHandler.ts → Real-time events

7. Database:
   src/config/database.ts → Supabase connection

Installation Order:
==================

1. Run: ./install.sh (or install.bat on Windows)
2. Edit: .env (add Supabase credentials)
3. Run: npm run dev
4. Test: http://localhost:3000/api/v1/health


Module Pattern (for new features):
==================================

To add a new module (e.g., Properties):

1. Create service:
   src/services/propertyService.ts

2. Create controller:
   src/controllers/propertyController.ts

3. Create routes:
   src/routes/v1/propertyRoutes.ts

4. Create validators:
   src/validators/propertyValidator.ts

5. Register routes:
   Update src/routes/v1/index.ts

6. Follow customer module as example!


API Versioning:
==============
